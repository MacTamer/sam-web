Section 1 — Overview

What Sam Is

Sam is a private, personal AI chatbot that lives on the internet. You own it completely — it is not shared with anyone, there is no public signup, and your conversations are stored in your own private database.

Think of it like your personal version of ChatGPT, but:

  • Only you can log in
  • Your data stays in your own accounts
  • You can customize Sam's personality and behavior
  • It runs 24/7 on a real URL you can open from your phone, laptop, or anywhere

How It Works (Simple Version)

  1. You open the website and log in with your username + password
  2. You type a message to Sam
  3. The message gets sent to a server (running on Vercel)
  4. The server saves your message, then asks OpenAI GPT-4o to respond
  5. The response streams back to your screen in real time, word by word
  6. Everything gets saved to your private database (Supabase)
  7. Your conversation history persists across devices and sessions

Section 2 — Tech Stack

Frontend

Next.js 16 (App Router) — The framework that builds the website. Handles both the pages you see AND the server-side API routes. "App Router" is the modern way Next.js organizes pages using folders.

React 18 — The UI library Next.js uses under the hood. Everything you see (chat bubbles, sidebar, buttons) is a React component.

TypeScript — A stricter version of JavaScript that catches errors before they happen. All .tsx and .ts files use TypeScript.

@ai-sdk/react v3 — Vercel's official library for building AI chat interfaces in React. Provides the useChat hook that manages streaming, message state, and sending. v3 is a major redesign — messages now use a parts array instead of plain content strings.

Backend

Next.js API Routes — The backend lives inside the same Next.js project under src/app/api/. Each folder is a different API endpoint. No separate server needed — Vercel runs these as serverless functions.

Vercel AI SDK v6 (ai package) — Core library for calling AI models and streaming responses. streamText() calls GPT-4o and streams the reply back token by token. TextStreamChatTransport connects the client's useChat to the server's text stream.

@ai-sdk/openai v3 — The OpenAI provider for the AI SDK. Wraps your OpenAI API key and lets you call openai('gpt-4o').

Database & Auth

Supabase — A hosted PostgreSQL database with built-in authentication. Stores conversations, messages, user profiles, and user settings. Handles login sessions via cookies and JWT tokens. Row Level Security (RLS) ensures each user can only see their own data.

@supabase/ssr + @supabase/supabase-js — Libraries for connecting to Supabase from Next.js. @supabase/ssr handles server-side session management correctly for Next.js 16.

Hosting & Version Control

Vercel — Hosts the entire app (frontend + backend). Connected to GitHub — every push to main automatically redeploys. Free plan handles personal use with no issues.

Git + GitHub — All code is stored in a private GitHub repo (MacTamer/sam-web). Vercel watches this repo and auto-deploys on every push.

Key Files Explained

src/proxy.ts — Runs on every request before the page loads. If you are not logged in and try to visit /chat, it redirects you to /login. Also refreshes your Supabase session token automatically. In Next.js 16, this file is named proxy.ts (not middleware.ts).

src/lib/prompts.ts — Takes your profile (name, interests) and settings (tone, directness, custom instructions) and builds the system prompt that Sam receives before every conversation. This is what makes Sam feel personalized.

src/app/api/chat/[convId]/route.ts — The core of the app. Receives your message, saves it to the database, fetches your conversation history, calls GPT-4o with your system prompt, and streams the response back word by word.

src/lib/supabase/server.ts — Critical detail: in Next.js 16, cookies() is async and must be awaited. This file handles that correctly. If you forget the await, you get a 404 on every page.

Section 5 — How the System Works (Flow)

Login Flow

1. You visit the live URL
2. proxy.ts checks: are you logged in? (reads session cookie)
3. No session found → redirected to /login
4. You type username + password → form POSTs to /api/auth/login
5. Server checks: does username match SAM_USERNAME env var? (case-insensitive)
6. Yes → server calls Supabase signInWithPassword() using SAM_USER_EMAIL
7. Supabase validates the password and returns a session
8. Session is stored in a cookie
9. You are redirected to /chat
Why username instead of email? The login screen only asks for a username — you never see your email. The server maps the username to your internal Supabase email behind the scenes. This is a private app so there is only one valid username.

Chat Flow

1.  You type a message and press Enter (or click send)
2.  ChatArea.tsx calls sendMessage({ parts: [{ type: 'text', text: '...' }] })
3.  TextStreamChatTransport sends POST to /api/chat/[convId]
    Body: { messages: [all current messages including the new one] }
4.  Server extracts the last user message from the messages array
5.  Server saves the user message to the 'messages' table in Supabase
6.  If this is the first message, server generates a title from the first 50 chars
7.  Server fetches full conversation history from Supabase (up to 60 messages)
8.  Server fetches your profile + settings from Supabase
9.  Server calls buildSystemPrompt() to create Sam's personality instructions
10. Server calls streamText() → OpenAI GPT-4o starts generating
11. Response streams back token by token to your browser in real time
12. When streaming finishes, onFinish callback saves Sam's reply to Supabase
13. Client's onFinish fetches the conversation to update the title in the sidebar

How Conversations Are Saved

  • Every user message is saved before the AI is called (always recorded)
  • Every assistant reply is saved after streaming completes via onFinish
  • When you switch conversations, the app fetches the full history from Supabase
  • Conversations are ordered newest-first in the sidebar

How Custom Instructions Work

  1. You open Settings and configure name, tone, interests, or custom instructions
  2. Settings are saved to the user_settings table in Supabase
  3. On every chat request, the server fetches your settings
  4. buildSystemPrompt() in prompts.ts weaves them into a system prompt
  5. That prompt is sent to GPT-4o before your messages, shaping how Sam responds

Section 7 — How to Run / Update Sam

Running Locally (for development and testing)

# 1. Open terminal and go to the project folder
cd C:\Users\coled\sam-web

# 2. Start the development server
npm run dev

# 3. Open your browser to:
http://localhost:3000

The local dev server uses your .env.local file for secrets. Changes to files update instantly without restarting. Stop it by pressing Ctrl+C in the terminal.

You do NOT need the terminal open for the live site to work. The live site runs on Vercel's servers 24/7. The terminal is only needed when you are making changes on your computer.

Deploying Updates to the Live Site

# 1. Make your code changes (with Claude's help in this folder)

# 2. Commit and push to GitHub
git add .
git commit -m "describe what you changed"
git push

# 3. Vercel auto-detects the push and redeploys (~60 seconds)

That is it. No manual deploy step needed. Every push to main = new live version.

Checking If a Deploy Succeeded

Go to vercel.com → sam-web → Deployments tab. Green checkmark = success. If it fails, click the deployment to see the build logs. Copy the red error text and bring it here to fix.

Changing Sam's Personality Without Code

Log into the live site → click your name in the sidebar → Settings. You can change name, tone, response style, interests, and custom instructions. Changes take effect on the next message you send.

Changing Your Login Password

Go to supabase.com → your project → Authentication → Users. Click on your user and reset the password from there.

Changing Your Login Username

Go to Vercel → sam-web → Settings → Environment Variables. Update SAM_USERNAME to the new username and redeploy.

Section 8 — Current Limitations

AreaCurrent StateImpact
Markdown rendering Sam's replies display as plain text. Bold, bullet points, and code blocks show raw symbols like **bold** Medium — responses are readable but unformatted
Response length Capped at 600 output tokens (~450 words) per response Medium — longer answers get cut off
Conversation titles Set to first 50 characters of your first message — not smart titles Low — functional but not elegant
Regenerate button Backend route exists but there is no button in the UI Low — can't retry a bad response without refreshing
Error messages If OpenAI fails mid-stream, no friendly error shown in the UI Low — rare but confusing when it happens
Conversation search No search bar — you scroll to find old conversations Low now, grows over time
File / image support Text only — Sam cannot read files or images Medium for power use cases
Multi-user Single-user by design — auth is hardcoded for one person Not a limitation for personal use
OpenAI cost scaling GPT-4o charges per token — heavy use will cost more Low for personal use (~$2–5/month typical)

Section 9 — Next Steps (Roadmap)

Quick Wins (easy to add)

#FeatureWhat to do
1 Markdown rendering Install react-markdown. Replace the <div className="msg-content">{text}</div> in ChatArea.tsx with <ReactMarkdown>{text}</ReactMarkdown>. Bold, lists, and code blocks will render properly.
2 Regenerate button Add a "Regenerate" button to the last assistant message in ChatArea.tsx. It already has a backend route at /api/conversations/[id]/regenerate — just wire it up.
3 Longer responses Change maxOutputTokens: 600 to 1500 or 2000 in src/app/api/chat/[convId]/route.ts. Will cost slightly more per message.
4 Custom domain Buy a domain at Namecheap or Porkbun (~$10/yr). Connect it in Vercel → Settings → Domains. Follow Vercel's DNS instructions.

Medium Improvements

#FeatureWhat to do
5 AI-generated titles After the first exchange, make a second AI call with a short prompt asking for a 4-6 word title. Save it to the conversation. More useful than truncated first message.
6 Code syntax highlighting Install react-syntax-highlighter. Pass it as a component to react-markdown to render code blocks with color.
7 Error handling UI Use the error field returned by useChat to show a friendly "Something went wrong" message with a retry button when the stream fails.
8 Conversation search Add a search input to the sidebar. Filter the conversations list client-side by title, or add a Supabase full-text search query for message content.

Bigger Features

#FeatureWhat to do
9 Switch AI models Add a model selector (GPT-4o, GPT-4o mini, Claude). Store the preference in user settings. In the chat route, read the setting and pass the chosen model to streamText(). The AI SDK supports multiple providers.
10 Sam's memory Create a memories table in Supabase. Let Sam save facts across conversations. On each request, fetch relevant memories and include them in the system prompt.
11 Image / file uploads Use Supabase Storage for files. Pass image URLs to GPT-4o vision. Requires UI changes (file input), storage setup, and multimodal model configuration.
12 Voice input Add a microphone button using the browser's Web Speech API. Transcribes voice to text and drops it into the input field. No extra services needed for basic transcription.
